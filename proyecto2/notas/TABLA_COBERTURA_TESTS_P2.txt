Resumen por test

1. tb_uc_mc_cb.vhd
- Caso: idle
  Estados cubiertos: Inicio
  Evidencia: ready = 1 sin peticion

- Caso: read hit
  Estados cubiertos: Inicio
  Evidencia: inc_r = 1, mux_output = "00"

- Caso: write hit
  Estados cubiertos: Inicio
  Evidencia: inc_w = 1, MC_WE1 = 1, Update_dirty = 1

- Caso: lectura de registro interno
  Estados cubiertos: Inicio
  Evidencia: mux_output = "10"

- Caso: escritura ilegal en registro interno
  Estados cubiertos: Inicio
  Evidencia: load_addr_error = 1, Mem_ERROR queda latcheada

- Caso: limpieza de Mem_ERROR leyendo x01000000
  Estados cubiertos: Inicio
  Evidencia: Mem_ERROR vuelve a 0 tras la lectura interna

- Caso: read a MD Scratch
  Estados cubiertos: Inicio, single_word_transfer_addr, Send_Addr, single_word_transfer_data
  Evidencia: bus read por palabra, mux_output = "01", last_word = 1

- Caso: write miss cacheable con write-around
  Estados cubiertos: Inicio, single_word_transfer_addr, Send_Addr, single_word_transfer_data
  Evidencia: inc_w = 1, inc_m = 1, MC_bus_Write = 1, MC_send_data = 1

- Caso: read miss cacheable limpio
  Estados cubiertos: Inicio, block_transfer_addr, read_block, block_transfer_data, bajar_Frame
  Evidencia: inc_r = 1, inc_m = 1, escritura secuencial de bloque y tags al final

- Caso: read miss cacheable con copy-back
  Estados cubiertos: Inicio, Send_ADDR_CB, write_dirty_block, CopyBack, block_transfer_addr, read_block, block_transfer_data
  Evidencia: inc_cb = 1, send_dirty = 1, Block_copied_back = 1 y fetch posterior del bloque nuevo

- Caso: error por ausencia de DevSel
  Estados cubiertos: Inicio, block_transfer_addr, read_block, fallo
  Evidencia: Mem_ERROR = 1

2. tb_uc_mc_event_counts.vhd
- Caso: secuencia formal de eventos
  Estados cubiertos: Inicio, single_word_transfer_addr, Send_Addr, single_word_transfer_data, block_transfer_addr, read_block, block_transfer_data, Send_ADDR_CB, write_dirty_block, CopyBack, bajar_Frame
  Evidencia: pulsos observados exactos
  Resultado: m = 3, w = 2, r = 3, cb = 1

- Utilidad:
  Este test no solo cubre funcionalidad, sino que demuestra que cada evento incrementa su contador exactamente una vez.
